## <div id="1">1. typing - Аннотации типов</div>
> Модуль предоставляет классы, функции и декораторы для поддержки аннотаций типов.
### Использование
- Повышение читаемости кода.
- Предоставление статической проверки типов.
### Примеры
Используем аннотации типов для указания ожидаемых типов аргументов и возвращаемого значения фукнции. `Union` позволяет указать, что переменная `b` может быть как целым числом, так и числом с плавающей точкой. `List[int]` указывает на ожидание списка целых чисел:
```python
from typing import List, Tuple, Union

def combine_values(a: int, b: Union[int, float]) -> Tuple[int, float]:
    result_sum = a + b
    result_product = a * b
    return result_sum, result_product

numbers_list: List[int] = [1, 2, 3]
result = combine_values(5, 2.5)

print(result)
print(numbers_list)
```


## <div id="2">2. pydoc - Генератор документации и онлайн-справочная система</div>
> Модуль автоматически генерирует документацию из модулей Python. Документация может быть представлена в виде страниц текста на консоли, отображена в веб-браузере или сохранена в файлах HTML. Для модулей, классов, функций и методов отображаемая документация получается из строки документации обьекта (\_\_doc__) и рекурсивно из его документированных членов. Если строка документации отсутствует, модуль пытается получить описание из блока строк комментариев чуть выше определения класса, функции или метода в исходном файле или в верхней части модуля.
### Использование
- Создание документации.
### Примеры
Запускает веб-сервер и открывает браузер с сгенерированной документацией:
```python
python -m pydoc -b
```
Получение справочной документации в консоли модуля, класса или функции:
```python
python -m pydoc module_name
```
Создание файла с текстовой документацией для указанного модуля:
```python
python -m pydoc -w module_name
```
Получение текстовой документации о модуле `math` в виде строки:
```python
help_info = pydoc.TextDoc().docmodule(math)
```


## <div id="3">3. doctest - Тест интерактивных примеров</div>
> Модуль предоставляет классы и функции для написания тестов внутри документации фукнций и модулей. Позволяет включать тестовые примеры в документацию и автоматически проверять их при запуске тестов.
### Использование
- Тестирование кода.
### Примеры
```python
def add(a, b):
    """
    This function adds two numbers.

    >>> add(2, 3)
    5
    >>> add(-1, 1)
    0
    >>> add(0.5, 0.5)
    1.0
    """
    return a + b

if __name__ == "__main__":
    import doctest
    doctest.testmod()
```


## <div id="4">4. 2to3 - Автоматический перевод кода Python 2 на 3</div>
> Модуль предоставляет инструмент для автоматической конвертации исходного кода Python2 в код Python3. Является устаревшим.
### Использование
- Облегчение процесса обновления проектов с Python2 на Python3, учитывая различия в синтаксисе и стандартной библиотеке между этими версиями.
### Примеры
Запуск из командной строки:
```
2to3 example.py
```


## <div id="5">5. unittest - Фреймворк модульного тестирования</div>
> Модуль предоставляет фукнции и классы для построения и запуска тестов. Основан на концепции модульного тестирования. Содержит модуль `unittest.mock`, который позволяет заменять части тестируемой системы макетными обьектами.
### Использование
- Модульное тестирование.
### Примеры
Тестовый пример создается путем создания подкласса `TestCase`. Три отдельных теста определяются методами, имена которых начинаются с букв `test`. 
```python
class TestStringMethods(unittest.TestCase):

    def test_upper(self):
        self.assertEqual('foo'.upper(), 'FOO')

    def test_isupper(self):
        self.assertTrue('FOO'.isupper())
        self.assertFalse('Foo'.isupper())

    def test_split(self):
        s = 'hello world'
        self.assertEqual(s.split(), ['hello', 'world'])
        with self.assertRaises(TypeError):
            s.split(2)

if __name__ == '__main__':
    unittest.main()
```
Настройка методов, которые выполняются перед и после каждого теста:
```python
class MyTestCase(unittest.TestCase):
    def setUp(self):
        # настройка ресурсов перед тестом
        pass

    def tearDown(self):
        # очистка ресурсов после теста
        pass
```
Создание макетов обьектов, изменение их поведения и изоляция кода при тестировании:
```python
from unittest.mock import Mock, patch

# Пример использования Mock класса
mock_obj = Mock()
mock_obj.method.return_value = 42
result = mock_obj.method()
print(result)  # Вывод: 42

# Пример использования patch() функции
def example_function():
    # какой-то код
    value = external_dependency()
    # какой-то код

with patch('module_name.external_dependency', return_value=5):
    result = example_function()
    print(result)  # external_dependency будет заменено на mock объект с возвращаемым значением 5

# Пример использования MagicMock
mock_list = MagicMock()
mock_list.append(10)
mock_list.extend([1, 2, 3])
print(len(mock_list))  # Вывод: 4, потому что mock_list имитирует поведение списка

```


## <div id="6">6. test - Пакет регрессионных тестов для Python</div>
> Модуль предоставляет регреционные тесты, модули `test.support` и `test.regrtest`. Все тесты следует писать с использованием модуля `unittest` или `doctest`.
### Использование
- Тестирование.
### Примеры
Функция test_add содержит простые утверждения, проверяющие корректность функции add. При запуске этого файла как скрипта с помощью python script.py, модуль test выполнит тесты и выдаст результаты:
```python
def add(a, b):
    return a + b

def test_add():
    assert add(3, 5) == 8
    assert add(-2, 7) == 5

if __name__ == '__main__':
    from test import testmod
    testmod()
```
